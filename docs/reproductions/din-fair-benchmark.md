# 七篇工业推荐论文的统一 DIN 基线

## 比较协议

- 数据：MovieLens-100K，按时间 leave-two-out，全物品库排序。
- 常规任务：相同的 180 users、280 items、validation/test 和候选屏蔽规则。
- 随机性：seeds 42、43、44；模型构建前固定 Python、NumPy 与 PyTorch seed。
- 统一基线：原始 DIN 的 candidate-aware local activation、Dice 和 CTR loss；固定 48 维、batch 48、100 steps。
- 指标：三次运行的 Hit@10、NDCG@10 均值；主表中的提升统一为 `(paper NDCG / DIN NDCG - 1) × 100%`。
- 跨域 KD：为保持 source/target 用户隔离，在同一 60-user target split 上重训并评估 DIN，不能与其余六篇的绝对值横向比较。

常规任务的统一 DIN 为 Hit@10 `0.04815`、NDCG@10 `0.02167`。内部消融仍保留，用于判断论文核心模块本身是否带来增量，但不再作为主结果。

## 解读边界

这是公开小数据上的统一工程基准，不等同于论文私有工业数据结论。DIN 与生成式召回、跨域蒸馏、多模态特征的训练目标不同；统一数据和评估可以比较最终排序质量，但不能消除模型规模、额外预训练数据和离线 teacher 成本的差异。因此主表回答“在当前公开协议下谁排得更好”，内部消融回答“论文新增模块是否有效”，两者必须同时阅读。
